# Ganglia 核心指南设计 (Core Guidelines)

> **状态**：初始设计
> **相关**：[架构](ARCHITECTURE.md), [记忆架构](MEMORY_ARCHITECTURE.md)

## 1. 简介
遵循“记忆即代码”和“通过提示词的可控性”原则，Ganglia 利用项目级的配置文件来定义其核心操作指令、安全规则和行为指南。该文件 `GANGLIA.md` 作为 Agent 的“核心指令”，确保在同一项目的不同会话中行为保持一致。

## 2. `GANGLIA.md` 文件
`GANGLIA.md` 文件是位于项目根目录的 Markdown 文档。它旨在对人类友好、受版本控制，并且易于被用户编辑以调整 Agent 的行为。

### 2.1 文件结构（示例）
```markdown
# Ganglia 核心指南

## 1. 操作强制指令 (Operational Mandates)
- 始终优先使用非阻塞 Vert.x 模式。
- 对于任何超过 3 个步骤的任务，使用 `todo_add` 进行分解。
- 绝不修改项目根目录以外的文件。

## 2. 安全与防护 (Security & Safety)
- 绝不记录或提交 API 密钥或机密信息。
- 在执行 `rm -rf` 或类似的破坏性命令前，必须请求确认。

## 3. 项目规范 (Project Conventions)
- 仅用于数据的类使用 Java 17 records。
- 遵循 `docs/ARCHITECTURE.md` 中定义的架构模式。
```

## 3. 系统集成

### 3.1 加载机制
`ContextEngine` 负责使用其 `ContextResolver` 加载 `GANGLIA.md` 文件。

- **路径**：根目录（工作目录）。
- **回退**：如果 `GANGLIA.md` 缺失，解析器将回退到一组硬编码的“默认指南”。
- **优先级**：
    - **操作强制指令**：优先级 2（硬性规则）。
    - **项目上下文**：优先级 3（规范）。

### 3.2 提示词注入
`GANGLIA.md` 的内容按标题解析，并由 `ContextComposer` 以相应的优先级注入到系统提示词中。这确保了安全指令在 Token 优化期间绝不会被修剪。

## 4. 与其他系统的关系

| 系统 | 作用 | 持久化 |
| :--- | :--- | :--- |
| **指南 (`GANGLIA.md`)** | 不可变（基本）的行为规则和项目标准。 | 项目级 (Git) |
| **记忆 (`MEMORY.md`)** | 不断演的事实、偏好和架构决策。 | 项目级 (Git) |
| **技能** | 领域特定的专业知识和工具（例如“AWS 专家”）。 | 模块级 |

## 5. 益处
- **透明度**：用户可以清楚地看到 Agent 遵循的规则。
- **可控性**：用户可以轻松修改 Agent 行为，而无需更改 Java 代码。
- **便携性**：指南随项目仓库一起移动。
