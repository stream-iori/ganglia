# Implementation Plan: Refactor Observation Architecture

**Branch**: `003-refactor-observation-arch` | **Date**: 2026-03-11 | **Spec**: [specs/003-refactor-observation-arch/spec.md](spec.md)
**Input**: Feature specification from `/specs/003-refactor-observation-arch/spec.md`

## Summary

Fix the "broken typewriter effect" and TTY stream loss in the WebUI by unifying the disparate macro and micro observation pipelines into a single, cohesive EventBus-driven architecture. This involves introducing an `ExecutionContext` to decouple tools/models from Vert.x, and refactoring `WebUIEventPublisher` to consume the unified event stream instead of acting as a synchronous loop observer.

## Technical Context

**Language/Version**: Java 17  
**Primary Dependencies**: Vert.x 5  
**Core Components Affected**: `StandardAgentLoop`, `WebUIEventPublisher`, `BashTools`, `AbstractModelGateway`, `TraceManager`  
**Communication Mechanism**: Vert.x EventBus (`ganglia.observations.*`)  
**Design Pattern**: Publisher-Subscriber (Pub-Sub), Context Object Pattern

## Proposed Solution

1.  **Introduce `ExecutionContext` and `ObservationDispatcher` Interfaces**:
    -   Create `work/ganglia/port/internal/state/ExecutionContext.java` containing `sessionId()`, `emitStream(chunk)`, `emitError(Throwable)`.
    -   Create `work/ganglia/port/internal/state/ObservationDispatcher.java` to handle unified publishing.

2.  **Implement Unified Dispatcher**:
    -   Create `DefaultObservationDispatcher` that implements both `ObservationDispatcher` and `AgentLoopObserver`.
    -   This dispatcher will receive ALL events (macro from the loop, micro from tools/models via `ExecutionContext`) and publish them to BOTH `Constants.ADDRESS_OBSERVATIONS_PREFIX + sessionId` AND `Constants.ADDRESS_OBSERVATIONS_ALL`.

3.  **Decouple Tools and Gateways**:
    -   Update `ToolDefinition.execute` and related interfaces to accept `ExecutionContext`.
    -   Refactor `BashTools` to use `context.emitStream()` instead of `vertx.eventBus().publish()`.
    -   Update `ModelGateway.chatStream` to accept `ExecutionContext` (or pass it a specific token consumer callback) so `AbstractModelGateway` no longer references `vertx.eventBus()`.

4.  **Refactor WebUI Event Ingestion**:
    -   Remove `AgentLoopObserver` implementation from `WebUIEventPublisher`.
    -   Change `WebUIEventPublisher` to consume the `Constants.ADDRESS_OBSERVATIONS_ALL` topic on the EventBus.
    -   When it receives an `ObservationEvent`, it translates it to a `ServerEvent` and publishes it to the WebSocket topic `ganglia.ui.ws.events` (and caching topic).

## Migration & Rollback
-   **Rollback**: The changes are localized to the observation reporting mechanism. If issues arise, reverting the git commit will immediately restore the previous synchronous `AgentLoopObserver` behavior.
-   **Testing**: We will run existing unit tests (e.g., `ConfigManagerTest`, `WebUIEventPublisherTest`) and manually verify the streaming behavior in the WebUI mock/demo mode to ensure tokens and TTY logs appear correctly.
